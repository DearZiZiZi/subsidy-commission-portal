# Сценарий демонстрации (Team Spirit · портал комиссии)

## Дашборд (`/`)

1. Откройте главную страницу. Убедитесь, что KPI показывают **полный портфель** (количество заявок, сумма в млрд ₸, средний балл, прогноз >60) — данные из `PORTFOLIO_STATS`, сгенерированные из CSV.
2. Прокрутите вниз до блока **«Техническая методология скоринга»**: анимированная схема пайплайна, горизонтальная диаграмма стратегических весов (подсказки по наведению), сравнение log1p / sqrt / **cbrt**, таблица источников данных.

## Оценка заявки (`/evaluate`)

1. Выберите направление и область из списков — строки совпадают с FastAPI `Literal` в `service.py`.
2. Отправьте форму. При сумме ≥ 100 млн ₸ проверьте бейдж **риск-штрафа** с фактическим вычетом баллов (10% от суммы компонентов). Для овцеводства в январе–апреле — бейдж **+7** сезонного бонуса.

## API для разработчиков (`/api-docs`)

1. Укажите базовый URL API (по умолчанию `http://127.0.0.1:8000`), нажмите **«Проверить соединение»** — ожидается ответ GET `/health` или `/`.
2. В блоке **«Живой запрос»** отредактируйте JSON и нажмите **«Отправить запрос»** — слева тело POST `/evaluate`, справа ответ с подсветкой синтаксиса; скопируйте готовый **cURL**.
3. Переключите вкладки **Python / JavaScript / cURL** для примеров интеграции; ознакомьтесь с таблицей схемы ответа и переменными `NEXT_PUBLIC_*`.

## Шортлист и пакетная обработка

Кратко покажите фильтры шортлиста и загрузку CSV на странице пакетной обработки — без изменений сценария, если бизнес-логика не затрагивалась.
