# Портал комиссии: описание страниц

Краткое назначение каждого маршрута приложения `subsidy-commission-portal` (Next.js App Router).

---

## `/` — Дашборд (аналитика)

**Назначение:** обзор портфеля заявок для комиссии: KPI и графики без вызова отдельного aggregate-API.

**Как работает:** берётся демонстрационный набор `SYNTHETIC_APPLICANTS` из `src/data/sample-results.ts` (50 синтетических заявок). Считаются: число заявок, сумма запросов в млрд ₸, средний балл, доля заявок с баллом выше 60 (прогноз «одобрения»). Recharts строит гистограмму баллов, горизонтальные бары по направлениям, линейный тренд по месяцам, сетку областей со средним баллом. При `NEXT_PUBLIC_DEMO_MODE=true` показывается баннер демо-режима.

---

## `/evaluate` — Оценка одной заявки

**Назначение:** ввод полей заявки и расчёт скоринга через FastAPI `POST /evaluate` (или мок в демо-режиме).

**Как работает:** форма на React Hook Form + Zod (`meritFormSchema`): ID, направление, норматив, цель, сумма, область, месяц, район. Отправка вызывает `evaluateMeritScore` из `src/lib/api.ts`. Результат: анимированный круговой индикатор, бейдж уровня, радар и таблица по 5 компонентам, текст объяснения, при необходимости бейджи сезонного бонуса (овцы, янв–апр) и штрафа (≥100 млн ₸). Кнопки: добавить результат в Zustand-шортлист, экспорт PDF по одному заявителю.

---

## `/shortlist` — Шортлист комиссии

**Назначение:** работа комиссии с отфильтрованным портфелем и отдельным списком отобранных кандидатов с учётом бюджета.

**Как работает:** таблица строится по отфильтрованным строкам из `SYNTHETIC_APPLICANTS` (фильтры в Zustand: направления, области, диапазон баллов, месяц, топ N). TanStack Table: сортировка, раскрытие строки с детализацией компонентов и объяснением. Строки можно добавлять/убирать из **шортлиста** (persist в `localStorage`). **Бюджет:** редактируемый лимит минус сумма `requested_amount` по строкам в шортлисте. Экспорт: Excel по шортлисту, PDF заключения комиссии (асинхронно, со встроенным шрифтом для кириллицы). Флаг «!» помечает заявку на ручную проверку (`manualReviewIds`).

---

## `/batch` — Пакетная обработка

**Назначение:** массовый скоринг из CSV по шаблону колонок.

**Как работает:** загрузка CSV → разбор `parseApplicantCsv` (`src/lib/csv.ts`), предпросмотр первых строк. Кнопка запуска обрабатывает валидные строки с **ограничением параллелизма 10** (`runWithProgress` в `src/lib/batch-queue.ts`): результаты появляются по мере готовности. Ошибки по строкам показываются отдельно. После завершения доступны выгрузка CSV и Excel.

---

## `/methodology` — Методология и прозрачность

**Назначение:** аудит: формула, источники, веса направлений, ограничения модели.

**Как работает:** статическая страница с интерактивными ползунками (числовые инпуты) для пяти компонентов, чекбоксы штрафа 10% и бонуса +7, пересчёт итогового балла на клиенте. Таблица стратегических весов, список ссылок на нормативные источники, текст про эвристику технологического уровня и планируемый апгрейд (LLM).

---

## `/settings` — Настройки

**Назначение:** язык интерфейса (русский / қазақша) и справка по переменным окружения.

**Как работает:** переключатель языка пишет выбор в `localStorage` и обновляет контекст `I18nProvider`. Кратко отображаются `NEXT_PUBLIC_API_URL` и `NEXT_PUBLIC_DEMO_MODE` (только подсказка, без редактирования в UI).

---

## Общая оболочка

**Layout:** боковая навигация (сворачивается), верхняя панель с переключателем темы (светлая/тёмная), контент с отступами. Шрифты UI: DM Sans / DM Mono. Уведомления: Sonner.

---

## Экспорт PDF и кириллица

Встроенные шрифты **jsPDF** (Helvetica и т.п.) поддерживают в основном **Latin-1**. Текст на русском/казахском без встраивания TTF отображается как «кракозябры» (mojibake); цифры и латиница (`YES`) остаются читаемыми. В проекте перед генерацией PDF подключается **Noto Sans** из `public/fonts/NotoSans-Regular.ttf` (`addFileToVFS` / `addFont`), таблица `jspdf-autotable` использует то же семейство шрифтов.
